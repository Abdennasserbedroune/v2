When I set out to build an AI chatbot assistant, my goal was clear: create a multilingual chatbot that could intelligently answer questions about my professional profile while maintaining production-grade security, performance, and reliability. What started as a simple chat interface evolved into a comprehensive system with 40+ Q&A entries, streaming responses, and robust error handling. This article documents the entire journey—from architectural decisions to implementation challenges.

Project Overview and Architecture

The AI Agent is built as a Next.js application leveraging the Groq API for AI inference. I chose Groq specifically because it offers free-tier access with approximately 30 requests per minute, making it ideal for a personal project that needed cloud-based LLM capabilities without infrastructure overhead.

The architecture follows a clear separation of concerns. The frontend handles user interactions and displays streaming responses, while the backend manages API calls, context orchestration, security validations, and rate limiting. All communication happens through a single /api/chat endpoint that implements Server-Sent Events (SSE) for real-time streaming.

Core Technology Stack

Next.js 13+ TypeScript Groq API Mixtral-8x7b-32768 Zod Jest Playwright Server-Sent Events

Why Groq?

Groq provides free-tier access with ~30 requests/minute, cloud-based inference with no infrastructure management, fast response times with the Mixtral model, and reliable uptime for production use.

The Profile Database: 40+ Bilingual Q&A Entries

One of the most time-consuming aspects was building a comprehensive profile database. I created over 40 question-answer pairs covering my skills, projects, experience, education, and professional background. Each entry exists in both English and French, enabling true bilingual support.

The profile data lives in /public/data/profile.json and follows a structured format that the system can easily parse and search. This approach allows the chatbot to surface relevant information based on user queries through intelligent context orchestration rather than relying solely on the AI model's general knowledge.

Profile Structure

{
  "id": "unique-identifier",
  "question": "What are your technical skills?",
  "answer": "I have strong expertise in...",
  "language": "en",
  "category": "skills",
  "keywords": ["react", "nodejs", "typescript"]
}

Database Benefits

  • Prevents AI hallucination with factual profile data
  • Enables context-aware responses without fine-tuning
  • Easy to update without code changes
  • Supports multilingual content natively

Intelligent Context Orchestration

Rather than sending all 40+ profile entries to the AI on every request, I implemented a relevance scoring system that automatically surfaces the most pertinent information. This was crucial for keeping API costs low and improving response quality.

The Orchestration Process

User Query ↓ Keyword Extraction ↓ Match Against Profile Keywords ↓ Calculate Relevance Scores ↓ Select Top 5 Entries ↓ Inject into System Prompt ↓ AI Generates Response

For example, when a user asks "What projects have you built?", the system identifies keywords like "projects" and "built," then retrieves profile entries tagged with those terms. The AI receives these specific entries as context, enabling accurate responses without hallucination.

Example System Prompt

You are a helpful AI assistant with access to profile information...

Available Profile Context:
1. Q: What are your skills?
   A: I have strong skills in React, Node.js, TypeScript...
2. Q: What projects have you built?
   A: I've built several full-stack applications...

Instructions:
- Use the profile context above to answer questions
- If context lacks information, ask clarifying questions
- Maintain a professional yet friendly tone

Security Implementation: Defense in Depth

Security was paramount throughout development. I implemented multiple layers of protection following industry best practices for AI Agent security.

Input Validation and Sanitization

Every request passes through Zod schema validation before processing. Messages must be between 1-4096 characters, roles must be either 'user' or 'assistant', and the last message must always come from the user. This prevents malformed requests from reaching the AI.

Beyond validation, all message content undergoes sanitization to remove control characters, invalid UTF-8 sequences, and potential prompt injection attempts. I strip characters in ranges 0x00-0x08, 0x0B-0x0C, 0x0E-0x1F, and 0x7F that could be used maliciously.

API Key Protection

Critical Security Measures

  • API key stored exclusively in environment variables
  • Never logged or exposed in error messages
  • Never sent to the frontend
  • All API communication happens server-side only

Rate Limiting

I implemented a token bucket rate limiter that restricts each IP address to 30 requests per minute, matching Groq's free tier limits. The limiter uses a fair distribution algorithm and includes automatic cleanup of inactive IP tracking data.

CORS Configuration

Proper CORS headers ensure the API only accepts requests with correct Content-Type validation. Only POST and OPTIONS methods are permitted, preventing unauthorized cross-origin access.

The Streaming Pipeline: Real-Time Responses

Implementing streaming responses was one of the most technically challenging aspects. Users expect modern AI Agents to display text as it's generated, not wait for complete responses. I built this using Server-Sent Events (SSE), which maintains a persistent connection for unidirectional server-to-client streaming.

Complete Pipeline Flow

Client Request ↓ Rate Limit Check (per IP) ↓ Input Validation (Zod) ↓ Message Sanitization ↓ Context Orchestration (relevant profile entries) ↓ Groq API Call (with retry + backoff) ↓ SSE Stream (chunk by chunk) ↓ Error Handling (graceful fallback) ↓ Connection Cleanup

SSE Response Format

// Successful streaming
data: {"type": "content", "data": "Hello"}
data: {"type": "content", "data": " from"}
data: {"type": "content", "data": " Groq"}
data: {"type": "done"}

// Error response
data: {"type": "error", "error": "User-friendly message", "code": "ERROR_CODE"}

The frontend consumes this stream using the EventSource API, parsing each data chunk and appending it to the chat interface with a typing effect. This creates the illusion of the AI "thinking" and responding in real-time.

Robust Error Handling and Retry Logic

Network issues, API timeouts, and service outages are inevitable. I built comprehensive error handling specifically tailored to Groq's API behavior.

Groq-Specific Error Handling

Error Type Status Code User Message Recovery Strategy
Invalid API Key 401 Authentication failed Check GROQ_API_KEY
Rate Limited 429 Rate limit exceeded Retry after delay
Timeout 408 Request timed out Auto-retry with backoff
Service Down 503 Service unavailable Auto-retry with backoff

Exponential Backoff Strategy

For transient errors like timeouts or service unavailability, the system automatically retries with exponential backoff. The initial retry happens after 1 second, then 2 seconds, then 4 seconds, with a maximum of 3 retries configurable via GROQ_MAX_RETRIES.

Retry Logic Benefits

  • Prevents overwhelming the API during outages
  • Gives temporary issues time to resolve
  • Improves user experience with automatic recovery
  • Reduces manual intervention requirements

Bilingual Support: Seamless Language Switching

Supporting both English and French required more than just translating UI strings. The system needed to detect the user's language, retrieve appropriate profile entries, and maintain language consistency throughout the conversation.

I implemented automatic language detection that analyzes the user's first message to determine their preferred language. Once detected, the system filters the profile database to only retrieve entries in that language. This prevents mixing English and French responses mid-conversation.

Language Detection Flow

  1. User sends first message
  2. System analyzes message content for language indicators
  3. Sets conversation language preference
  4. Filters profile database by language
  5. Instructs AI to respond in detected language
  6. Maintains consistency for entire conversation

The language parameter is included in every API request, allowing the AI to respond appropriately. Users can explicitly switch languages at any time by sending a message in their preferred language, and the system adapts immediately.

Challenges and Solutions

Challenge: Managing Groq Free Tier Limits

The 30 requests per minute limit required careful rate limiting implementation.

Solution: Token bucket algorithm with per-IP tracking, which fairly distributes the quota and prevents abuse while accommodating legitimate burst traffic.

Challenge: Preventing Prompt Injection

Malicious users could attempt to inject commands into messages to manipulate the AI's behavior.

Solution: Comprehensive input sanitization that removes control characters and validates message structure, combined with a well-designed system prompt that clearly defines the AI's role and boundaries.

Challenge: Streaming Response Reliability

SSE connections can break unexpectedly due to network issues or user navigation.

Solution: Graceful error handling with automatic reconnection attempts, cleanup of server-side resources when connections close, and clear error messages to users when streams fail.

Challenge: Context Window Management

The Mixtral model has a 32,768 token context window. With long conversations, tokens could be exhausted.

Solution: Limit the number of profile entries injected (top 5 only), implement conversation history trimming for very long chats, and monitor token usage to warn before hitting limits.

Challenge: Bilingual Consistency

Early implementations sometimes mixed languages mid-conversation.

Solution: Made language detection explicit rather than automatic per-message, filtered profile data by language before context injection, and instructed the AI explicitly about language consistency in the system prompt.

Deployment and Production Readiness

The application is deployed on Vercel, leveraging Next.js native support for seamless deployment. Environment variables are securely configured through Vercel's dashboard, ensuring the API key never appears in source code or logs.

Production Checklist

  • ✅ GROQ_API_KEY set in environment
  • ✅ npm run build succeeds without errors
  • ✅ npm run lint passes with zero warnings
  • ✅ npm run type-check validates all TypeScript
  • ✅ npm run test passes all test suites
  • ✅ Streaming tested with real API key
  • ✅ Rate limiting verified under load
  • ✅ Error logs verified (no sensitive data exposure)
  • ✅ SSL/HTTPS enabled via Vercel
  • ✅ CORS configured for production domain

Production Optimizations

  • Next.js automatic code splitting for faster page loads
  • Server-side rendering for initial chat interface
  • Static generation for documentation pages
  • Edge runtime support for minimal latency

Key Learnings and Future Improvements

Building this AI Agent taught me invaluable lessons about production AI systems:

Key Learnings

  • Security cannot be an afterthought - It must be designed in from the beginning
  • Streaming greatly improves perceived performance - Even when total response time is similar
  • Comprehensive error handling builds user trust - When things go wrong
  • Testing saves time in the long run - By catching bugs early
  • Rate limiting is essential - For free-tier API usage

Future Enhancements

  • Implementing conversation memory across sessions using a database
  • Adding support for additional languages beyond English and French
  • Integrating web search for questions outside the profile database
  • Implementing user authentication for personalized conversations
  • Adding analytics to understand common queries and improve the profile database

Conclusion

Building this production-ready AI Agent was a comprehensive exercise in full-stack development, API integration, security implementation, and user experience design. The result is a system that reliably answers questions about my professional profile in two languages, handles errors gracefully, prevents abuse through rate limiting, and provides a smooth streaming experience that feels modern and responsive.
