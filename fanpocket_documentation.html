Football is more than just a sport in Morocco—it's a way of life, a source of national pride, and a unifying force that brings communities together. When I set out to build Fanpocket, my vision was to create a digital companion for Moroccan football fans that would connect them with their local football culture, help them discover stadiums, track their favorite teams, and stay updated on matches in real-time.

This article documents the complete journey of building Fanpocket: a production-ready Progressive Web App (PWA) that combines modern web technologies, mobile-first design, and a deep understanding of the Moroccan football landscape. From architectural decisions to implementation challenges, this is the story of bringing a football fan experience to life on the web.

Project Vision and Purpose

The Problem

Moroccan football fans face several challenges when it comes to engaging with their local football culture:

  • Fragmented Information - Stadium locations, team schedules, and match information are scattered across multiple sources
  • Limited Digital Presence - Many local teams and stadiums lack comprehensive online presence
  • Language Barriers - Content is often available in only one language (French, Arabic, or English)
  • Mobile Access - Fans need instant access to information while on the go, often in areas with poor connectivity

The Solution: Fanpocket

Fanpocket is designed as a comprehensive digital companion that addresses these challenges through:

Core Objectives

  • Stadium Discovery - Interactive map showing all football stadiums across Morocco with detailed information
  • Team Tracking - Follow your favorite teams, view their schedules, and get match notifications
  • Match Updates - Real-time scores, upcoming fixtures, and match results
  • Offline Access - Progressive Web App functionality allowing offline browsing of cached content
  • Multilingual - Full support for English, French, and Arabic to serve all Moroccan football fans
  • Community Features - Connect with other fans, share experiences, and build the football community

Target Audience

Fanpocket is built for:

  • Local Fans - Moroccans who follow domestic leagues and local teams
  • Travelers - Tourists and visitors who want to experience Moroccan football culture
  • Diaspora - Moroccan expats staying connected with home country football
  • International Fans - Global football enthusiasts interested in African football

System Architecture and Design

Fanpocket follows a modern monorepo architecture with clear separation of concerns between client, server, and shared code. This architectural decision enables code reuse, consistent type safety, and streamlined development workflows.

High-Level Architecture

Client (Next.js 14+ PWA) ↓ API Gateway / Load Balancer ↓ Express.js REST API ↓ MongoDB Database ↓ External Services: - Firebase Cloud Messaging (Notifications) - Map Tile Providers (OpenStreetMap)

Architectural Benefits

  • Type Safety - Shared TypeScript types ensure consistency between frontend and backend

  • Independent Deployment - Client and server can be deployed separately

Technology Stack Deep Dive

Frontend Technologies

Next.js 14+ TypeScript React 18 Tailwind CSS shadcn/ui next-pwa next-i18next React Leaflet

Next.js 14 with App Router

I chose Next.js 14 for several compelling reasons. The new App Router provides better performance through React Server Components, improved routing with nested layouts, and built-in data fetching optimizations. The framework's server-side rendering capabilities are crucial for SEO while the static site generation features enable fast page loads.

PWA Implementation with next-pwa

The next-pwa package transforms the Next.js app into a full-fledged Progressive Web App. This enables:

  • Installation on user devices (appears like a native app)
  • Offline functionality through service workers
  • Background sync for data updates
  • Push notifications via Firebase Cloud Messaging
  • Caching strategies for optimal performance

Styling with Tailwind CSS and shadcn/ui

Tailwind CSS provides utility-first styling that keeps the bundle size small and development speed high. Combined with shadcn/ui components, I get accessible, well-designed UI components that can be customized to match the Moroccan football aesthetic—bold reds and greens inspired by the national flag.

Backend Technologies

Express.js TypeScript MongoDB Mongoose JWT Firebase Admin

Express.js API Server

Express.js provides a lightweight, flexible foundation for the REST API. With TypeScript, the entire backend maintains compile-time type safety, catching errors early in development. The middleware architecture allows clean separation of concerns for authentication, validation, and error handling.

MongoDB with Mongoose

MongoDB's document-based structure perfectly fits the varied data shapes in Fanpocket—stadiums, teams, matches, users, and notifications all have flexible schemas that can evolve. Mongoose provides schema validation, relationship management, and a clean query interface.

JWT Authentication

JSON Web Tokens enable stateless authentication, crucial for a distributed system. Users authenticate once and receive a token that grants access to protected resources. This scales well and works seamlessly with the PWA's offline capabilities.

Development Tools

Docker Compose ESLint Prettier Husky commitlint

Development quality is maintained through automated tools: ESLint for code quality, Prettier for consistent formatting, Husky for git hooks, and commitlint for conventional commit messages. Docker Compose provides local MongoDB instances for development and testing.

Progressive Web App Implementation

Transforming Fanpocket into a PWA was one of the most critical architectural decisions. PWAs bridge the gap between web and native apps, providing app-like experiences while maintaining web accessibility.

Service Worker Strategy

The service worker handles all caching and offline functionality. I implemented a cache-first strategy for static assets (images, CSS, JavaScript) and a network-first strategy for dynamic content (match data, team information).

// next.config.js
const withPWA = require('next-pwa')({
  dest: 'public',
  register: true,
  skipWaiting: true,
  disable: process.env.NODE_ENV === 'development',
  runtimeCaching: [
    {
      urlPattern: /^https:\/\/api\.fanpocket\.ma\/.*/i,
      handler: 'NetworkFirst',
      options: {
        cacheName: 'api-cache',
        expiration: {
          maxEntries: 50,
          maxAgeSeconds: 60 * 60 * 24 // 24 hours
        }
      }
    },
    {
      urlPattern: /\.(?:png|jpg|jpeg|svg|gif|webp)$/i,
      handler: 'CacheFirst',
      options: {
        cacheName: 'image-cache',
        expiration: {
          maxEntries: 100,
          maxAgeSeconds: 60 * 60 * 24 * 30 // 30 days
        }
      }
    }
  ]
});

module.exports = withPWA({
  // Next.js config
});

Monorepo Structure and Workspaces

Managing a full-stack application with separate client and server codebases can be challenging. I opted for a monorepo approach using npm workspaces, which provides significant development benefits.

Workspace Configuration

// package.json (root)

  "name": "fanpocket",
  "private": true,

    "client",

    "shared"

    "dev:client": "npm run dev -w client",

    "build": "npm run build --workspaces",
    "lint": "npm run lint --workspaces",

  }
}

Shared Types

One of the biggest advantages of the monorepo is shared TypeScript types. Both client and server reference the same type definitions, eliminating sync issues:

i18n Challenges

  • Number Formatting - Different decimal and thousand separators per locale
  • Date Formats - French uses DD/MM/YYYY, English uses MM/DD/YYYY
  • Pluralization - Arabic has complex plural rules
  • Font Loading - Arabic fonts require different loading strategy
  • Layout Shifts - RTL can cause layout issues if not tested thoroughly

Database Schema

MongoDB schemas defined with Mongoose:

// models/Stadium.ts
import mongoose from 'mongoose';

const stadiumSchema = new mongoose.Schema({
  name: { type: String, required: true, unique: true },
  city: { type: String, required: true },
  capacity: { type: Number, required: true },
  location: {
    type: { type: String, enum: ['Point'], default: 'Point' },
    coordinates: { type: [Number], required: true } // [lon, lat]
  },
  teams: [{ type: mongoose.Schema.Types.ObjectId, ref: 'Team' }],
  image: String,
  amenities: [String],
  createdAt: { type: Date, default: Date.now }
});

// Create geospatial index for location queries
stadiumSchema.index({ location: '2dsphere' });

export default mongoose.model('Stadium', stadiumSchema);

Authentication Middleware

JWT-based authentication protects user-specific endpoints:

// middleware/auth.ts
import jwt from 'jsonwebtoken';

export function authenticate(req, res, next) {
  const token = req.headers.authorization?.split(' ')[1];

  if (!token) {
    return res.status(401).json({ error: 'No token provided' });
  }

  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    req.user = decoded;
    next();
  } catch (error) {
    return res.status(401).json({ error: 'Invalid token' });
  }
}

Development Challenges and Solutions

Challenge: Offline Map Functionality

Problem: Map tiles require network requests. How to provide offline maps?

Solution: Implemented service worker caching for map tiles within Morocco's bounds. When online, tiles are cached aggressively. Offline users see cached tiles for previously visited areas.

Challenge: Real-Time Match Updates

Problem: Live scores require constant data updates, draining battery and data.

Solution: Implemented smart polling that increases frequency during active matches and decreases for pre/post-match states. Used WebSocket fallback for critical matches.

Challenge: Arabic RTL Layout

Problem: RTL breaks certain components (maps, charts, carousels).

Solution: Used Tailwind's RTL plugin with selective LTR override classes for components that shouldn't flip. Extensive testing with native Arabic speakers.

Challenge: PWA Installation Prompts

Problem: Users don't know they can install the app.

Solution: Built custom install prompt that appears after user engagement (visiting 3+ pages). Shows installation benefits specific to football fans.

Challenge: Image Optimization

Problem: Stadium and team images consuming too much bandwidth.

Solution: Next.js Image component with automatic WebP conversion, lazy loading, and responsive sizing. Service worker caches optimized images.

Challenge: Geolocation Permissions

Problem: Many users deny location access, breaking "Near Me" feature.

Solution: Made location optional. If denied, show city-based fallback. Clear messaging about why location improves experience.

The Next.js PWA is deployed on Vercel, which provides:

  • Automatic HTTPS

MongoDB Atlas hosts the production database with:

Key Learnings and Future Roadmap

Technical Learnings

  • PWAs are Production-Ready - With proper caching strategies, PWAs perform as well as native apps
  • Monorepos Simplify Full-Stack Development - Shared types eliminate entire classes of bugs
  • i18n Must Be Planned Early - Retrofitting multilingual support is painful
  • Maps Are Complex - Seemingly simple features (clustering, offline tiles) require deep optimization
  • TypeScript Pays Off - The upfront investment in types prevents runtime errors

Product Learnings

  • Offline-First Design Matters - Many Moroccan areas have spotty connectivity
  • Language Preference is Personal - Users switch between languages frequently
  • Visual Design Conveys Culture - Red and green color scheme resonates with Moroccan users
  • Mobile-First is Essential - 95% of users access via mobile

Future Roadmap

  • Social Features - User profiles, friend lists, match check-ins
  • Ticket Integration - Direct ticket purchasing for matches
  • Live Commentary - Text-based live updates for major matches
  • Video Highlights - Short-form match highlights
  • Fantasy League - Moroccan football fantasy game
  • Podcast Integration - Football podcasts and analysis

Conclusion

Building Fanpocket has been an incredible journey of technical growth and cultural connection. Creating a Progressive Web App that serves Moroccan football fans required not just modern web development skills, but also deep understanding of the local football landscape, linguistic diversity, and connectivity challenges.

The project demonstrates that with careful architecture—monorepo structure, shared TypeScript types, PWA capabilities—it's possible to build production-grade applications that rival native apps in user experience while maintaining web accessibility and instant updates.

Project Achievements

  • Full Progressive Web App with offline functionality
  • Trilingual support (English, French, Arabic) with RTL layout
  • Interactive stadium map with React Leaflet
  • Real-time match updates and push notifications
  • Monorepo architecture with shared TypeScript types
  • Production deployment on Vercel and Railway
  • MongoDB Atlas database with geospatial queries
  • Comprehensive CI/CD pipeline

The complete source code is available at GitHub. I encourage developers interested in PWAs, Next.js, or monorepo architecture to explore the codebase and adapt the patterns to their own projects.

For Moroccan football fans, Fanpocket represents a new way to engage with the beautiful game—one stadium, one team, one match at a time.
